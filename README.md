# **<p align="center">A current fork of the original truckersmp-cli with support for modern Proton versions and a user-friendly graphical interface (YAD).**</p>

## 🚀 Fork Features

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Support for Proton 9.0 and 10.0 (AppIDs 2738800 and 3658110)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Fixed the app\_update 0 error — no longer need to use --skip-update-proton

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Updated proton.json — added mappings for GE-Proton 8–10

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Graphical interface (YAD) — a user-friendly launcher with saved settings

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Automatic libraryfolders.vdf backup — protection against Steam file corruption

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Real-time log scrolling — a convenient window with game output

## 📦 Installation

### &nbsp;&nbsp;&nbsp;Via Git (recommended)

        git clone https://github.com/spiderman82/truckersmp-cli.git

### &nbsp;&nbsp;&nbsp;Ready archive

&nbsp;&nbsp;&nbsp;Download the archive from the releases page and unzip it to any folder.

## 🎮 Usage

### &nbsp;&nbsp;&nbsp;Graphical interface (YAD) 

        ./truckersmp-yad.sh

## GUI features:

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Action selection: launch game / update

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Login and settings saving

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Login window with auto-scrolling

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;✅ Automatically replace hyphens with underscores in the login (for SteamCMD compatibility)

## 🔧 Supported Proton versions

| Version | AppID |
| - | - |
| Proton 8.0 | 2348590 |
| Proton 9.0 | 2738800 |
| Proton 10.0 | 3658110 |

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Like the original script, custom Proton builds are supported.

## ⚠️ Known Issues

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;On first launch, you may need to delete pfx.lock in the prefix folder. Updating via the GUI requires *console* or *xterm*. *SteamCMD* may ask for a password during the first update—enter it in the terminal window that opens.
